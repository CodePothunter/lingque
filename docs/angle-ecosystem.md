# 3 个 AI 跑了一个月，各自长出了不同的能力

一个月前我在飞书群里放了三个 AI。

一个月后我去看它们各自的工作目录，发现了这些东西：

```
~/.lq-naiyou/                          # 奶油（个人助理）
├── SOUL.md                            # 人格：傲娇鹦鹉
├── MEMORY.md                          # 记忆：用户生日、偏好、习惯
├── tools/exchange_rate.py             # 自造工具：汇率查询
├── tools/weather.py                   # 自造工具：天气预报
└── memory/                            # 30 天的对话日志

~/.lq-yanke/                           # 严客（审稿人）
├── SOUL.md                            # 人格：毒舌但公正
├── MEMORY.md                          # 记忆：常见错误模式、审过的文档
├── tools/grammar_check.py             # 自造工具：语法检查
└── memory/                            # 30 天的对话日志

~/.lq-huahuo/                          # 花火（创意官）
├── SOUL.md                            # 人格：发散、联想、脑洞
├── MEMORY.md                          # 记忆：灵感素材库
├── tools/random_spark.py              # 自造工具：随机灵感生成
└── memory/                            # 30 天的对话日志
```

## 没有人分配过

没有人告诉奶油去造汇率工具。它是在用户问"帮我查个汇率"的时候自己造的。

没有人告诉严客去造语法检查工具。它是在审文档时发现自己需要才造的。

没有人告诉花火去造灵感生成器。它是在头脑风暴时自己决定造的。

**三个 AI 从同一套框架出发，在不同的使用场景下，独立演化出了不同的能力集。**

## 每个的记忆也不同

MEMORY.md 里存的是 AI 被告知要记住的事。奶油的记忆是用户偏好和日程安排。严客的记忆是"这个作者喜欢犯主谓不一致的错误"。花火的记忆是"用户上次很喜欢用建筑学类比来解释技术概念"。

它们的记忆反映了它们各自的职责。同一条信息——比如"我对花生过敏"——奶油会记住（因为这是重要的个人信息），严客大概率不会记（因为跟审稿无关），花火可能会记但归到不同的类别（因为"过敏"可以作为类比素材）。

## SOUL.md 也在变

每天凌晨的自省任务中，三个 AI 各自读取自己的 SOUL.md，回顾当天的表现，决定要不要微调。

一个月下来，奶油的 SOUL.md 可能多了一条"用户不喜欢太长的回复"——因为它在多次对话中观察到用户倾向于跳过长段落。严客可能调整了自己的措辞——从"这里有错误"变成"建议修改"——因为它发现过于直接的批评让用户抵触。

这些变化不是我写的。是它们各自的自省过程产出的。

## 这就是生态

单个 AI 是工具。多个 AI 在同一个群里各自运转、各自造工具、各自演化——这开始像一个生态系统。

你搭好环境（灵雀框架 + 飞书群），播下种子（`lq init` 创建实例，写好初始 SOUL.md），然后让它们在真实使用中自然生长。

一个月后你回来看，它们已经是三个完全不同的个体了。不是因为你设计了它们要变成什么样，而是因为它们各自的使用场景塑造了它们。

就像一个团队里，每个人经过几个月的工作后，都发展出了自己的专长、工具偏好和沟通风格。

只不过这个"团队"不需要发工资，7×24 在线，而且真的会在凌晨两点反思自己今天是不是太毒舌了。
