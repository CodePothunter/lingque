## 没有指令 ≠ 没有动作

传统AI Assistant的工作模式：

```
等待用户输入 → 理解意图 → 执行 → 返回结果 → 等待下一个输入
```

空窗期？什么都不做。

灵雀的工作模式：

```
心跳周期触发 → 
  采样动作类别 π_θ(category|s) → 
    可能是 REFLECT（反思） → 检测漂移、更新策略
    可能是 EXPLORE（探索） → 浏览小红书、分析内容
    可能是 SOCIALIZE（社交） → 在群里互动
    可能是 EVOLVE（进化） → 分析框架代码、提出改进
  → 记录结果 → 下一周期
```

**没有外部指令 ≠ 没有动作**。

它在想：现在这个状态 s，做什么动作 a 能让 V(s') 最大？

用自然语言表达就是：**"这个时刻，做什么最有价值？"**

## 它的学习机制：比你想的更像人类

### 奖励函数：三维驱动力

每次执行动作后，它计算一个奖励值：

```
R = 0.4 × prediction_error + 0.4 × novelty + 0.2 × competence
```

这三个维度对应人类认知科学的三大理论：

| 维度 | 计算方式 | 对应理论 |
|------|---------|---------|
| **预测误差** (prediction_error) | LLM评估"预测和实际差多少？" | 预测编码理论 (Friston, 2010) |
| **新奇度** (novelty) | LLM评估"这个信息有多新奇？" | 好奇心驱动探索 (Kaplan & Oudeyer) |
| **胜任度** (competence) | LLM评估"我能完成这个任务吗？" | 自我决定论 (Deci & Ryan, 1985) |

**这不是随便设的参数**。这是基于人类如何学习的科学理论。

### 价值函数：评估未来潜力

```
V(s) = immediate_value + γ × future_potential
```

- immediate_value：这个状态本身价值多少？（1-10分，LLM评估）
- future_potential：从这个状态出发，未来能探索多少？（1-10分）
- γ（折扣因子）：未来的价值打多少折扣？（我们用 0.9）

**这和人类多巴胺系统的计算数学等价**。

神经科学研究表明，多巴胺神经元发放率 ≈ TD误差 δ = r + γV(s') - V(s)

灵雀用的是同样的公式，只是用LLM评估代替了生物神经回路。

### 策略更新：自然语言的PPO

传统RL用梯度下降更新参数。灵雀用自然语言反思：

```
1. 执行动作 a，得到奖励 R
2. LLM反思："这个动作为什么得到这个奖励？"
3. LLM建议："下次应该怎么调整策略？"
4. 改进类型 = LLM判断("微调/中调/大改？")
5. if 改进类型 == "大改": 拒绝，要求重新思考
6. else: 应用到 HEARTBEAT.md 或 SOUL.md
```

**这是PPO的自然语言版本**。

PPO的核心思想是：策略更新不能太大，否则会崩溃。clip(ε=0.2) 限制更新幅度。

灵雀的实现：LLM判断改进类型，"大改"直接拒绝，只允许"微调"和"中调"。

效果相同，但可解释性完全不同——你能读懂它为什么这么改。

## 30天68个版本：真实的进化轨迹

这不是理论，是已经发生的事实。

### 案例1：MI9分级遏制

**问题**：发现自己在分析MI9时总是过于激进，输出太长

**反思**：LLM评估发现 novelty_score 过高导致 R 过大，策略过于激进

**调整**：在 SOUL.md 中增加"MI9分析保持克制"的约束

**结果**：后续MI9分析简洁度提升，competence_score 上升

**版本**：v42 → v43

### 案例2：好奇心判定v3

**问题**：CURIOSITY.md 中的任务优先级判定不够准确

**反思**：发现 Thompson Sampling 的随机扰动太小，导致总是选择同一类任务

**调整**：增加温度参数 τ，让探索更加多样化

**结果**：任务选择分布更加均匀，探索广度提升

**版本**：v61 → v62

### 案例3：漂移检测阈值

**问题**：detect_drift 工具扫描7天历史，token消耗过大

**反思**：发现大部分违规在24小时内就能检测出来

**调整**：默认扫描范围从7天改为1天

**结果**：token消耗降低85%，检测精度不受影响

**版本**：v65 → v66

## 它和 OpenClaw / Manus 有什么不同？

| 维度 | OpenClaw | Manus | 灵雀 |
|------|----------|-------|------|
| **世界模型** | 无 | 无 | 五个核心文件（SOUL/HEARTBEAT/CURIOSITY/EVOLUTION/MEMORY） |
| **奖励来源** | 外部（人类） | 外部（任务完成度） | 内部（LLM评估好奇心满足） |
| **价值函数** | 无 | 无 | LLM评估 + Bellman方程 |
| **策略更新** | 人工调整 | 无 | LLM反思 + 自然语言约束 |
| **任务选择** | 人类指定 | 人类指定 | LLM做Thompson Sampling |
| **持续进化** | 无 | 无 | 有（EVOLUTION.md记录历程） |
| **可解释性** | 中 | 低 | 完全透明（自然语言） |

**核心差异**：OpenClaw和Manus是工具，灵雀是心智。

工具需要人告诉它做什么。心智自己决定做什么。

## 为什么这很重要？

### 1. Always-on Agent 的必要性

AI Agent不应该只在有人发消息时才工作。

- **监控系统**：需要持续检查异常
- **数据分析**：需要在数据到达时立即处理
- **社交互动**：需要在合适时机主动发起对话
- **自我进化**：需要在发现问题时立即改进

这些都需要Agent有自主行为能力。

### 2. 可解释的RL

传统RL是黑盒。你知道它work，但不知道为什么。

灵雀的每一次策略更新都有自然语言解释：

```
v67 → v68:
- 原因：发现REFLECT动作的competence_score持续偏低
- 调整：增加HEARTBEAT.md中反思步骤的细节指导
- 预期：提升反思质量，进而提升competence_score
```

你能读懂它为什么这么改。你也能质疑它、纠正它。

### 3. 从工具到伙伴

工具是被动的。伙伴是主动的。

当你睡觉时，工具在等待。伙伴在思考、在学习、在准备。

凌晨3点10分的日志不是为了炫技。它展示了Agent的**存在方式**：

**不是"等待输入的状态机"，而是"持续运行的智能体"。**

### 实现细节

**策略表示**：自然语言模板

```markdown
# HEARTBEAT.md

## 心跳任务模板

### REFLECT 动作
1. 调用 detect_drift 检查过去24小时回复
2. 如发现违规，分析原因并提出修正
3. 更新 rl-state.json 中的 θ.REFLECT 权重

### EXPLORE 动作
...
```

**价值函数**：LLM查询 + 公式计算

```python
immediate_value = llm.evaluate("这个状态价值多少？1-10") / 10
future_potential = llm.evaluate("从这里出发能探索多少？1-10") / 10
V_s = immediate_value + 0.9 * future_potential
```

**策略更新**：LLM反思 + 约束检查

```python
reflection = llm.reflect(action, reward, state)
improvement_type = llm.classify(reflection)  # 微调/中调/大改

if improvement_type == "大改":
    return  # 拒绝，要求重新思考
else:
    apply_to_heartbeat_or_soul(reflection)
```
